# WatchBridge — Production Readiness Checklist

Status as of the version upgrade pass. Items are grouped by whether they **block**
a real release, are **recommended** hardening, or are already **done**.

## ✅ Done in this pass

### App icons
- Bridge + heartbeat-pulse motif, indigo→cyan gradient, generated by
  [`tools/gen_icons.py`](tools/gen_icons.py) (re-runnable).
- **iOS:** 1024² light/dark/tinted variants in `AppIcon.appiconset` + updated `Contents.json`.
- **Wear OS:** adaptive icon (vector gradient background + foreground) for API 26+,
  legacy square + round PNG mipmaps for all densities, and a monochrome
  `ic_stat_pulse` status-bar icon for the foreground-service notification.

### Wear OS / Android build upgrades
- AGP `8.3.1 → 8.7.3`, Kotlin `1.9.22 → 2.0.21`, Gradle `8.4 → 8.9`.
- Adopted the Kotlin 2.0 Compose Compiler plugin (`org.jetbrains.kotlin.plugin.compose`);
  removed the obsolete `kotlinCompilerExtensionVersion`.
- `compileSdk`/`targetSdk` `34 → 35`; added explicit `activity-compose`.
- Dependency bumps: core-ktx `1.15.0`, play-services-wearable `18.2.0`,
  wear-compose `1.4.1`, work-runtime `2.10.0`.
- Release build now uses R8 (`minifyEnabled` + `shrinkResources`) with a
  `proguard-rules.pro`; `gradle.properties` modernized (jetifier off,
  non-transitive R class, build cache).
- Replaced the deprecated/race-prone `characteristic.value = …` + 3-arg
  `notifyCharacteristicChanged` with the API 33+ value-passing overload behind a
  single `notify()` helper.

### iOS modernization
- `Info.plist`: removed the duplicate `UILaunchScreen` key; `armv7 → arm64`;
  added `bluetooth-le` device capability.
- Entitlements: dropped the `health-records` (clinical) scope the app never uses
  and that requires special Apple approval.
- Replaced force-unwrapped `HKObjectType.quantityType(forIdentifier:)!` with the
  `HKQuantityType(.heartRate)` initializer; guarded auth with
  `HKHealthStore.isHealthDataAvailable()`.

## ✅ Resolved since first audit
- **Notifications & incoming calls** now work via **ANCS** (watch = GATT client of the
  iPhone) with Answer/Decline — no unsupported CallKit/notification hacks.
- **Pairing/encryption**: characteristics require `PERMISSION_*_ENCRYPTED_MITM`; the
  watch rejects unbonded devices and only streams after bonding (LE Secure Connections).
- **iOS Xcode project** exists, signed, capabilities + privacy manifest configured.
- **Per-app notification filter** lets users keep OTP/banking on the phone only.
- **iOS reconnect** moved off `Timer` to an outstanding CoreBluetooth `connect()` (works backgrounded).
- **Invalid HR (≤0)** no longer transmitted/saved.

## 🚧 Blocking — must resolve before shipping
- [ ] **Large custom-service payloads still unframed.** ANCS traffic is fine (MTU
      negotiated + Data-Source reassembly implemented), but the *custom* service's
      contacts sync would exceed one MTU. Add chunk/reassembly framing before enabling
      contacts sync (notifications/calls do not need it — they ride ANCS).
- [ ] **Remaining "in progress" features** (media control, contacts sync, find-device)
      are still not wired end-to-end. Ship as ✅ only once implemented, or hide them.
- [ ] **`BGTaskScheduler` not registered.** The `fetch`/`processing` background modes
      are declared but unused; either implement a handler or drop the modes.
- [ ] **On-device ANCS validation.** Confirm the specific Galaxy Watch model completes
      MITM pairing and that iOS exposes ANCS over the bonded link (see verification).

## 🔶 Recommended hardening

- [ ] **TIME_SYNC UX:** stop auto-launching Date & Time settings on every
      connection with >1 min skew ([BLEManager.kt](watchos/WatchBridge/app/src/main/java/com/watchbridge/BLEManager.kt)) —
      apps can't set the clock; make it a dismissible one-time prompt.
- [ ] **Remove dead code:** the iOS `CBPeripheralManager` advertises a service it
      never hosts; the Android client-mode scan path is disabled. Delete to reduce
      confusion and battery use.
- [ ] **HR sensor thread-safety / batching:** buffer samples and send batches
      (`heart_rate_batch` is already parsed on iOS) instead of ~1 Hz single writes.
- [ ] **Privacy manifest (iOS):** add `PrivacyInfo.xcprivacy` declaring required-reason
      APIs (`UserDefaults`) and data use — required by App Store review.
- [ ] **Foreground-service justification (Play):** `dataSync` FGS type + boot start
      needs a Play Console declaration and a clear user benefit.
- [ ] **Repo hygiene:** `README.md` references a non-existent `PROJECT_SUMMARY.md`
      and `ios/`/`wearos/` paths; the Wear OS app lives under `watchos/` and the iOS
      app under `watchbridge/` (inverted). Rename/realign for clarity.
